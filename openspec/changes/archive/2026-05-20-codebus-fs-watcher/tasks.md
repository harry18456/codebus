- [x] 1.3 在 `watcher.rs` 加 Per-Path Debounce Window（spec: Per-Path Debounce Window；design D2）：`HashMap<PathBuf, Instant>` + tokio sleep，同 path 200ms 內合併、不同 path 各自獨立 timer。行為：atomic-rename 序列 50ms 內被合併為單一 Tauri emit；不同 path 不互相延遲。驗證：`watcher_tests.rs` 加 `atomic_rename_emits_one_event` 與 `distinct_paths_debounce_independently`（mock event source、assert emit count + 時序）。
- [x] 1.4 在 `watcher.rs` 實作 Event Catalog And Payloads（spec: Event Catalog And Payloads；design D5）：path → event 分類與七種 emit（`wiki-list-changed` / `wiki-page-changed { path }` / `goals-changed` / `goal-run-changed { run_id }` / `quiz-changed` / `quiz-attempt-changed { slug, id }` / `vault-list-changed`）；`run_id` 自 `events-<slug>.jsonl` 抽取、`slug` 自 `<slug>/<id>.md|.progress.json` 抽取。行為：每種 path pattern 對應正確 event name 與 payload。驗證：`watcher_tests.rs` 為每種 event 加一個 case（共 7 個）assert `event_name` 與 payload 結構。
- [x] 1.5 在 `lib.rs` `setup` hook 啟動 Lobby Watcher（spec: Lobby Watcher）：永久監聽 `~/.codebus/app-state.json`，emit `vault-list-changed`；handle 存進 `AppRuntimeState`。行為：app 啟動後 watcher active；外部修改檔案會觸發 emit；app 關閉時自動釋放。驗證：`watcher_tests.rs` 加 `lobby_watcher_emits_on_app_state_change`（mock path、寫入、assert event 400ms 內 emit）。
- [x] 1.6 在 `watcher.rs` 加 Excluded Watch Scope（spec: Excluded Watch Scope）：對 `<vault>/.codebus/raw/`、`<vault>/.codebus/CLAUDE.md`、`~/.codebus/config.yaml`、以及 `.lock`、`.git`、`**/.obsidian/` 模式不 emit 任何 event。行為：這些路徑 raw event 在 classify 階段被丟棄、不進 debounce、不 emit。驗證：`watcher_tests.rs` 加 `excluded_paths_emit_nothing`（對列出的每個排除路徑各觸發 raw event、assert 零 emit）。
- [x] 1.7 在 `watcher.rs` 處理 Watcher Startup Failure Surfaces Loudly（spec: Watcher Startup Failure Surfaces Loudly；design D6）：`notify::Watcher::new` 失敗時 emit `vault-watcher-error { vault_path, reason }`、不重試、之後該 vault 完全靜默（不 emit 其他 watcher event）。行為：watcher 構造失敗 → error event 一次；之後該 vault 任何 raw event 都不 emit。驗證：`watcher_tests.rs` 加 `watcher_new_failure_emits_error_once`（注入失敗 constructor、assert 1 次 error + 0 次其他 event）。
