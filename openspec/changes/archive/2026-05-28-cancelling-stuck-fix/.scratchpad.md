## Task 1.3 — idle binaries

Local dev environment confirmation:
- `where sleep` → `C:\Program Files\Git\usr\bin\sleep.exe`（Git for Windows）
- `where timeout` → `C:\Windows\System32\timeout.exe`（native Windows）AND `C:\Program Files\Git\usr\bin\timeout.exe`

Decision: tests SHALL pick binary by `cfg!(target_family = ...)`：

- Unix（`cfg(unix)`）：`std::process::Command::new("sleep").arg("30")` — POSIX standard, always on PATH.
- Windows（`cfg(windows)`）：`std::process::Command::new("cmd").args(["/c", "timeout", "/t", "30", "/nobreak", ">", "nul"])` — `timeout.exe` is part of Windows since XP, always present on `C:\Windows\System32` which is on default `PATH`. CI runners covered.

For streaming child（Task 2.2）: use a tiny inline child that echoes a line, sleeps 50ms, echoes again. Easier path = invoke a `cmd /c` chain on Windows or `sh -c` on Unix; cleaner path = use a small fake `AgentBackend` that returns parse events synthetically. Lean toward fake backend route: avoid spawning real child for streaming test; the silent-child test still needs a real child (kill semantics).
